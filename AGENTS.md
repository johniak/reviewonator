# Project Rules

## Code quality

- Design code for testability from the start.
- Prefer the smallest practical implementation while keeping the code clean, readable, and maintainable.
- Use well-established libraries for solved problems instead of reinventing them.

## Product and design

- The application must be visually polished and provide a pleasant user experience.
- All user-facing content and the entire application interface must be in English.

## Testing

- All application behavior must be covered by tests.
- Prefer tests that exercise real behavior and real integrations.
- Use mocks only as a last resort when no practical alternative exists, never merely for convenience.
